# Dive to Soul 项目文档

## 项目概述

Dive to Soul 是一款基于 Godot 引擎开发的卡牌回合制角色扮演游戏。玩家将在游戏中探索不同的节点，与敌人战斗，收集卡牌，提升能力，最终挑战 boss。

## 核心系统

### 1. 游戏流程系统

游戏采用节点式流程，玩家需要依次通过以下节点：
- start：游戏开始
- dialogue：对话场景
- reward：奖励场景
- battle_normal：普通战斗
- event：事件场景
- rest：休息场景
- battle_boss： boss 战斗

游戏管理器（`game_manager.gd`）负责控制游戏流程，跟踪当前节点位置，并提供重置游戏进度的功能。

### 2. 战斗系统

战斗系统是游戏的核心，采用回合制卡牌战斗机制：

- **玩家回合**：
  - 重置能量
  - 抽取卡牌（5张）
  - 玩家可以选择使用卡牌
  - 结束回合

- **敌人回合**：
  - 敌人根据能量进行攻击
  - 计算伤害并应用到玩家

- **战斗结束条件**：
  - 敌人 HP 降至 0：玩家胜利
  - 玩家 HP 降至 0：玩家失败

### 3. 卡牌系统

游戏中有以下类型的卡牌：

| 卡牌 ID | 名称 | 类型 | 费用 | 认知 | 效果 |
|--------|------|------|------|------|------|
| strike | 斩击 | 攻击 | 1 | 1 | 对一名敌人造成 2 点伤害 |
| bless | 祝福 | 增益 | 2 | 3 | 恢复 5 点存在值 |
| break | 瓦解 | 减益 | 2 | 2 | 使敌人获得 1 层虚弱 |
| relief | 释怀 | 运营 | 0 | 3 | 恢复 1 点费用 |
| resonance | 认知共振 | 攻击 | 2 | 3 | 造成等同当前认知负荷值的伤害 |

### 4. 属性系统

#### 玩家属性
- **HP**：生命值，降至 0 则游戏失败
- **SAN**：理智值，降至 0 则玩家发疯
- **能量**：每回合重置，用于使用卡牌
- **认知负荷**：使用卡牌会增加，超过上限会导致 HP 减半
- **虚弱**：减少造成的伤害

#### 敌人属性
- **HP**：生命值，降至 0 则敌人被击败
- **SAN**：理智值
- **能量**：每回合可使用的行动次数
- **虚弱**：减少造成的伤害

### 5. 场景系统

游戏包含多个场景，每个场景对应不同的游戏阶段：

- **标题场景**：游戏开始界面
- **地图场景**：显示游戏节点和进度
- **对话场景**：展示游戏剧情和对话
- **战斗场景**：进行卡牌战斗
- **事件场景**：随机事件和选择
- **休息场景**：恢复 HP 和 SAN
- **奖励场景**：获得卡牌和奖励
- **结果场景**：显示游戏结果

## 游戏玩法

1. **开始游戏**：从标题场景进入游戏
2. **探索节点**：在地图场景中选择节点
3. **战斗**：与敌人进行卡牌战斗
4. **收集卡牌**：通过奖励场景获得新卡牌
5. **管理资源**：合理使用能量和认知负荷
6. **挑战 Boss**：最终挑战强大的 Boss

## 技术实现

- **引擎**：Godot 4.6
- **渲染**：Forward Plus
- **物理**：Jolt Physics
- **脚本**：GDScript

## 开发指南

### 运行游戏

1. 安装 Godot 4.6 或更高版本
2. 打开 `project.godot` 文件
3. 点击运行按钮启动游戏

### 添加新卡牌

1. 打开 `assets/data/cards.json`，按照现有格式添加新卡牌的 JSON 字典数据（必须包含 `id`, `name`, `type`, `cost`, `cognition`, `desc`）。

2. 在 `scripts/battle_manager.gd` 中找到卡牌点击处理方法（`on_enemy_target_clicked` 或 `on_card_clicked`）。

3. 增加对应卡牌 ID 的分支判定，并编写该卡牌的具体战斗效果逻辑（例如 `play_fireball()`）。

### 添加新敌人

1. 打开 `assets/data/enemies.json`，添加新的敌人数据节点（设置 `id`, `name`, `max_hp`, `max_energy` 等字段）。

2. 在需要生成该敌人的地方，直接调用 `enemy.setup_from_db("新敌人ID")` 即可，无需编写额外的初始化代码。

### 添加新事件与对话

1. 打开 `assets/data/events.json`。
2. **添加剧情对话：**建立带有 `lines` 数组的数据节点，`dialogue_scene.gd` 会自动解析并逐句显示。
3. **添加随机事件：**建立带有 `options`（选项）的数据节点。可直接在 JSON 中配置 `max_hp_change`、`extra_energy_change` 等数值字段，事件系统将自动读取并对玩家属性生效。

### 添加新场景

1. 在 `scenes` 目录中创建新场景文件
2. 在 `scripts` 目录中创建对应的脚本文件
3. 在 `game_manager.gd` 的 `map_nodes` 数组中添加新节点类型
