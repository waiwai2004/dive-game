# Dive-Game - 程序组 Git 协作与 Godot 开发指南

兄弟们，咱们的底层框架仓库已经建好了！为了保证咱们 4 个人同时写代码不打架、不互相覆盖进度，请大家严格按照以下流程操作。花 5 分钟看完，能帮咱们以后省下无数个修 Bug 的通宵！

## 🌟 核心军规（切记！）

- **绝对、绝对、绝对不要直接在 master（主干）分支上写代码和提交！**
- 所有的开发都要在自己的个人分支上进行，写完后提交 PR（合并请求），由组长审核后合并进主干。

## 🛠️ 第一阶段：初次拉取项目（仅需做一次）

1. 在你的电脑上找个空间够的纯英文路径文件夹（比如 D:\GodotProjects\）。
2. 在该文件夹空白处右键 -> 打开 Git Bash 或命令行终端。
3. 输入以下命令拉取代码：
   ```bash
   git clone https://github.com/waiwai2004/dive-game.git
   ```
4. 拉取成功后，打开 Godot 引擎，点击"导入(Import)"，选中刚刚拉下来的文件夹里的 project.godot 文件即可打开项目。

## 🔄 第二阶段：日常开发循环（每天/每个新功能都要走一遍）

每次你要开始做一个新功能（比如写卡牌拖拽、写怪物 AI），请严格执行以下 4 步：

### 👣 第 1 步：同步最新代码并建新分支

在写任何代码前，先确保你的主干是最新的，然后从主干拉一个新分支出来：

```bash
# 1. 切换到主干
git checkout master

# 2. 拉取组长合并过的最新进度
git pull origin master

# 3. 创建并切换到你的专属开发分支（名字自己起，见名知意）
# 比如前端写UI：git checkout -b feature/ui-card
# 比如写怪物AI：git checkout -b feature/monster-ai
git checkout -b 你的分支名字
```

### 👣 第 2 步：在 Godot 里愉快地写代码

现在你已经在自己的分支上了，随便怎么写都不会影响别人！

当你完成了一个小阶段（比如代码能跑通了），就可以提交保存了：

```bash
# 1. 把改动添加到暂存区（注意 add 后面有个空格和点）
git add .

# 2. 提交改动并写明做了什么
git commit -m "完成：实现了卡牌拖拽的基础动画"
```

> 提示：一天可以 commit 多次，当成游戏存档就行

### 👣 第 3 步：把你的分支推送到云端

功能做完后，把你的分支推送到 GitHub：

```bash
git push -u origin 你的分支名字
```

> 注意：最后是你的分支名字，千万不要写 master！

### 👣 第 4 步：在 GitHub 上提 PR（Pull Request）

1. 登录 GitHub 进入咱们的仓库主页。
2. 页面顶部会提示你刚推送了一个新分支，点击旁边绿色的 Compare & pull request 按钮。
3. 简单写一句你做了啥，然后点击 Create pull request。
4. 去群里吼一声："队长，我提 PR 了，去看看代码！"

队长验收无误后，会把你这部分代码合并进 master。合并后，你可以删掉这个旧分支，回到第 1 步接下一个任务！

## 💣 第三阶段：Godot 防冲突"三大纪律"

Godot 的场景文件（.tscn）是纯文本，多人同时改同一个场景百分之百会代码爆炸！为了各自安好，请遵守：

### 万物皆可实例化（保存为单独场景）：
- 比如你负责做【卡牌】，请新建一个 Card.tscn。
- 比如你负责做【怪物】，请新建一个 Monster.tscn。
- 每个人只在自己的测试场景（比如 UITest.tscn 或 AITest.tscn）里测自己的东西。最终由队长统一拼装到主场景里。

### 坚决不碰别人的文件：
- 负责 AI 的不要去改 UI 的脚本，需要交互就走**全局信号（Signal/Event）**或者等队长定义接口。

### 改 project.godot 必须群里报备：
- 如果你需要添加【全局单例（Autoload）】、【新的按键映射（Input Map）】或【安装插件】，这会修改项目核心配置文件。改之前一定要在群里吼一声！ 防止大家同时改配置导致项目崩溃。
